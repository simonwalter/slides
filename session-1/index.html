Basic theory

Single Responsibility Principle (SRP) in practice

/ HTML has no own module concept yet

/ CSS still global by default

/ JS modules in the browser are based on build systems

/ Breaking your code into parts does not document it

/ Building more "things" means more naming, which is hard

Two problems with SRP

/ Some help with classification and naming would be nice

/ Components geared towards the browser need non-trivial tech in the development setup
